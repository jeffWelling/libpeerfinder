Design

 

Theres a few things we want to keep in mind when we are designing this.  We want to make extensive use of plugins where ever possible.  It will be designed so that the swarms that it looks for are defined by plugins from the beginning, so all you have to do to be able to connect to a new swarm is add/write a plugin for it.  It comes with some desireable defaults, such as one to help with connecting to trackers, and one for DHTs for torrents.

Another area that will be modular like the swarm definitions is the methods that it uses to find peers.  It will of course come with several defaults such as broadcasting, specialized network scanning to identify hosts that you are close to topologically, and IRC, but will also allow for additional methods of locating peers to be defined by writing a plugin with a few specific hooks.

Additional design suggestions are welcome.
